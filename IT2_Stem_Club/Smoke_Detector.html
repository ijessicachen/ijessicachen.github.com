Arduino Smoke Detector

By: April Fools! ( Shiloh and Jessica )


Description

Our project is a homemade, shabby, flammable, inefficient replica of a modern type of fire alarm. This device is useful because it can save lives by detecting whenever there is a fire.

There are three types of fire alarms: ionization, photoelectric, and ionization/photoelectric. An ionization smoke alarm has an ionization chamber in it, and that's basically two plates with a voltage across them. This smoke alarm also has a radioactive source of ionizing radiation. These work together like this: the particles generated ionize the oxygen and nitrogen atoms of the air in the chamber. Basically, ionizing knocks electrons off of atoms. The electron without an atom has a negative charge, while the atom missing one electron has a positive charge. These get drawn to the plate with the opposite voltage. This all comes into play when smoke enters the ioniation chamber, The smoke particales neutralize the ions, and the smoke alarm detects the drop in currents between the plates so the alarm goes off. A photoelectric smoke alarm is more simple. It contains a photoresistor, and some sort of light source for the detecting process. A photoresistor is just a sensor that detects light. Basically you shine the light at or under the photoresistor. When the smoke enters the smoke detector it will refract the light so that if it was pointing at the photoresistor, there is no more light shining at it, and if you were pointing it under, the light will get detected by the photoresistor. Finally, the ionization/photoelectric alarm is a combination of both alarms. Honestly, these are the best because while ionization alarms respond faster to fast-flame fires, they are signficantly slower at detecting smoldering fires than photoelectric alarms.

We chose to make the photoelectric smoke alarm. Mainly this was because it's just so much easier to make, with the laser module and photoresistor module we already have, but it's also because ionization alarms have nuclear radiation in them. It's okay though, because the amount is minuscule, but in the bustling workplace we were in we didn't wat anyone accidentally inhaling it because then it would be a danger. Plus, we would actually have to get an ionization chamber and ionizing radiation. The problem with our choice though, was that it was incredibly difficult to make the smoke alarm only react to smoke and not also to the lights witht he low quality materials, and amateur construction skills we had it was incredibly difficult for us to make sure the photoresistor wouldn't react to the wrong thing.

Circuit Diagram

This whole circuit will be in a box, connected to a 9V battery as its power source. The box will have some tiny slits/a hole in the bottom letting in any smoke. Basically the laser module will be pointing under the photocell, but when the smoke enters it well scatter the light beam and some of it will shine on the pohotocell module. When is lands on the photocell module a really annoying buzzer is programmed to go off. Of course, this is why the lights are so annoying because the code can accidentally mistake the lights/sunlight as the light that the smoke refracted.

int PCellValue = 0; // Photoresistor Value

int laserPin = 12;// Digital Pin for the laser module

int piezoPin = 8;// Digital pin for the piezo buzzer

void setup() {
Serial.begin(9600); // Begin the serial moniter. 9600 bits per second
pinMode(laserPin, OUTPUT); // The laser is the output, meaning it can provide some energy to other circuits
digitalWrite(laserPin, HIGH); // this basically turns the laser on
}

void loop() {

Our code is basically a bunch of variables, and setting up the sensors. After that we basically just have one if statement telling the program what to do if light enters the photocell. We hope the program will perceive the right thing as smoke.

The only special thing about our code is the function tone();.This function needs two arguments, but can also take three.
